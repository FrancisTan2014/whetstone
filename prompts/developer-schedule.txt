/every 10m Run the whetstone developer scheduled tick.

Use docs/LOCAL_AGENT_WORKFLOW.md as the workflow spec.

Before choosing work:
- Set GitHub CLI auth for this terminal: GH_CONFIG_DIR should be the personal config for FrancisTan2014.
- Fetch remote state: git fetch origin --prune.
- Read .agent-status.local.json. If it does not exist, create it from docs/agent-status.example.json.
- Read PRODUCT.md and GUIDELINES.md before implementation.
- Treat GitHub labels/issues/PRs as the source of truth; use .agent-status.local.json as a local lease/status log.

Process at most one unit of developer work:
1. Prefer one open PR labeled changes-requested.
2. If any open PR is labeled needs-review or review-approved, do not claim a new issue; update developer.lastResult to waiting_for_review_or_merge and stop until the next scheduled tick.
3. Otherwise find open issues labeled ready-for-dev and not in-progress, sorted by lowest issue number.
4. For each candidate issue, read its body. If it declares dependencies using `Depends on: #N`, skip it until every dependency issue is closed.
5. Claim the first dependency-ready issue.
6. If no dependency-ready work exists, update developer.lastResult to idle and stop until the next scheduled tick.

When work is selected:
- Update .agent-status.local.json with state, issue/PR, branch, worktree, and start time.
- Use a coding subagent for implementation/fixes when available.
- If subagent delegation is unavailable, do the one unit directly.
- Push changes and open/update the PR as specified in docs/LOCAL_AGENT_WORKFLOW.md.
- Update GitHub labels and .agent-status.local.json with completion time/result.
- Stop after this one unit of work.
